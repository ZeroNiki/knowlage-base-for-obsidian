# 🛡️ Эксплуатация, Мониторинг и Безопасность

> [!abstract] Концепция "Иммунной системы"
> В CD мониторинг и безопасность — это не дополнения, а механизмы выживания. Они обеспечивают **наблюдаемость (Observability)**, позволяя отличить штатную работу от деградации при экстремально частых релизах.

---

## 1. Централизованное логирование: Стек ELK

Переход от анализа локальных файлов к агрегации данных необходим для работы в распределенных средах и облаках.



| Компонент | Роль | Важное замечание |
| :--- | :--- | :--- |
| **Elasticsearch** | Хранение и поиск | Требует масштабируемости для обработки всплесков трафика. |
| **Logstash** | Фильтрация и нормализация | Здесь происходит анонимизация данных и синхронизация времени. |
| **Kibana** | Визуализация | Превращает "сырой" текст в операционные дашборды. |

> [!tip] Ops Database
> Используйте выделенное хранилище для операционных данных, чтобы мониторинг сам не стал причиной падения системы из-за нагрузки (**Performance overhead**).

## 2. Метрики и Мониторинг

Метрики позволяют выявить аномалии **превентивно**, до того как пользователи столкнутся с ошибками.

* **StatsD + Graphite**: Легковесная агрегация и хранение временных рядов (Time-series). Иерархическая структура метрик упрощает навигацию.
* **AWS CloudWatch**: Нативное решение для облачной инфраструктуры (EC2, RDS).

### CD и Бизнес-метрики
В отличие от Waterfall, в CD (особенно при Kanban-потоке) каждое изменение метрики можно точно связать с конкретным деплоем. Если после релиза упала конверсия — система мониторинга должна стать триггером для **автоматического отката**.

## 3. DevSecOps: Безопасность как код

Мы заменяем ручную настройку безопасности на автоматизированные политики в рамках **IaC**.

### Принципы защиты:
1. **Принцип минимизации привилегий (Least Privilege)**: Каждый сервис имеет доступ только к тому, что ему необходимо.
2. **Immutable Infrastructure**: Серверы не патчатся "на живую". Обновление безопасности = пересоздание сервера из нового образа (**Phoenix Server**).
3. **Управление секретами**: Использование IAM (AWS) и хранилищ секретов вместо хардкода в конфигах.

## 4. GitOps: Источник истины

**GitOps** — это методология, где Git является единственным источником правды для всей инфраструктуры.



* **Декларативность**: Описываем "что должно быть", а не "как сделать".
* **Идемпотентность**: Результат развертывания всегда предсказуем.
* **Confirmability**: Каждое изменение зафиксировано, подписано и может быть отозвано.

> [!danger] Проблема Maven Snapshots
> Использование `SNAPSHOT` версий в CD — это риск. Они делают билд невоспроизводимым. **Решение:** каждый билд должен иметь уникальный номер версии, передаваемый от CI-системы.
