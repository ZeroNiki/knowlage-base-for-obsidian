# ️ Инфраструктура как код (IaC): Методологии и Инструменты

> [!quote] Философия архитектора
> IaC — это не просто скрипты, а дисциплина, устраняющая «серверы-снежинки» и превращающая эксплуатацию в предсказуемый инженерный процесс.

---

## 1. Фундаментальные парадигмы

### Декларативный vs Императивный подход
| Критерий | Императивный (Bash, CLI) | Декларативный (Terraform, Puppet) |
| :--- | :--- | :--- |
| **Фокус** | "Как сделать" (последовательность команд) | "Что должно быть" (целевое состояние) |
| **Идемпотентность** | Сложно достичь | Гарантирована инструментом |
| **Дрейф конфига** | Высокий риск | Автоматическое исправление |

### Модели доставки: Push vs Pull



* **Push (Ansible):** Управляющий узел "толкает" конфиг через SSH. 
    * *Плюс:* Не нужны агенты (Agentless), легкий старт.
* **Pull (Puppet, Chef):** Агенты на серверах сами запрашивают конфиг у мастера.
    * *Плюс:* Идеально для Auto-scaling и огромных парков серверов.

## 2. Ландшафт инструментов

### Provisioning (Фундамент)
**Terraform** — управляет топологией (сети, инстансы, БД как сервисы).
* **HCL (HashiCorp Configuration Language)**: Декларативный язык.
* **State File**: Хранит текущее состояние инфраструктуры (критически важный артефакт).
* **Providers**: Плагины для AWS, GCP, Azure, vSphere.

### Configuration Management (Обустройство)
| Инструмент | Язык | Особенности |
| :--- | :--- | :--- |
| **Ansible** | YAML | Лаконичность, работа по SSH, низкий порог входа. |
| **Puppet** | DSL (Ruby) | Строгая декларативность, управление сложными зависимостями. |
| **Chef** | Ruby | Максимальная гибкость через Recipes и Cookbooks. |
| **SaltStack** | YAML/Python | Сверхскорость через шину ZeroMQ (Master/Minion). |

## 3. Инженерная дисциплина и Тестирование

### Качество и проверка
* **Flyway / Liquibase**: IaC для баз данных. Версионирование схем вместо ручных правок.
* **ChefSpec**: Unit-тестирование (симуляция ресурсов без создания VM).
* **Serverspec**: Интеграционное тестирование (проверка реального состояния портов и пакетов по SSH).

### Стратегия "Сервер-Феникс"
> [!important] Phoenix Server & Immutable Infrastructure
> Мы никогда не латаем старый сервер. При любом изменении старый инстанс уничтожается, и создается новый из проверенного кода. Это гарантирует отсутствие накопленных аномалий.

## 5. Lean-контекст: Сокращение потерь
Внедрение IaC напрямую коррелирует с принципами Lean:
1.  **Устранение Waste**: Быстрое развертывание сред сокращает время простоя.
2.  **Traceability**: Git как единственный источник истины для всей инфраструктуры.
3.  **Feedback Loop**: Быстрое тестирование изменений инфраструктуры в CI-пайплайне.
