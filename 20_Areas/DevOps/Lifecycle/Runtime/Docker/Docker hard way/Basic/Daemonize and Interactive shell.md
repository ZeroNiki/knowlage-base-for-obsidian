# Режимы запуска контейнеров: Detached vs Interactive

В Docker режим запуска определяет, как жизненный цикл процесса и его потоки ввода-вывода (I/O) связаны с текущей сессией терминала.

### 1. Режим Detached (`-d`)

Запускает контейнер в фоновом режиме. После выполнения команды Docker Engine возвращает идентификатор контейнера (UUID), и процесс продолжает выполнение независимо от текущего терминала.

* **Механика:** Стандартные потоки (STDOUT, STDERR) перенаправляются в систему логирования Docker (доступны через `docker logs`).
* **Сценарии использования:** Сервисы с длительным циклом работы (веб-серверы, базы данных, очереди сообщений).
* **Управление:** Для остановки или удаления требуется явная команда `docker stop` или `docker rm`.

### 2. Режим Interactive + TTY (`-it`)

Комбинация двух флагов, обеспечивающая полноценное взаимодействие с процессом внутри контейнера в реальном времени.

* **`-i` (Interactive):** Поддерживает поток стандартного ввода (**STDIN**) открытым, даже если терминал не прикреплен. Это позволяет процессу принимать команды от пользователя.
* **`-t` (TTY):** Эмулирует виртуальный терминал (pseudo-TTY). Необходим для корректной работы интерактивных оболочек (`bash`, `sh`, `zsh`), обеспечивая отрисовку приглашения к вводу и поддержку управляющих последовательностей.

### 3. Сравнительная характеристика

| Характеристика | Detached (`-d`) | Interactive (`-it`) |
| --- | --- | --- |
| **Связь с терминалом** | Фоновая (фоновый процесс) | Прямая (интерактивная сессия) |
| **Потоки вывода** | Пишутся в логи на диске | Отображаются в терминале |
| **Завершение работы** | Контейнер работает при закрытии консоли | Выход из оболочки (`exit`) обычно останавливает контейнер |
| **Применение** | Production, серверные компоненты | Отладка, тестирование, ручное конфигурирование |

### 4. Дополнительные возможности

* **Переподключение:** К контейнеру, запущенному в режиме `-d`, можно подключиться позже с помощью команды `docker attach` или запустить в нем новую оболочку через `docker exec -it <container_id> /bin/bash`.
* **Режим `--rm`:** Часто используется вместе с `-it`, чтобы автоматически удалять контейнер сразу после завершения интерактивной сессии, предотвращая накопление "мусорных" слоев в системе.
