# Инициализация Docker в Linux

Процесс запуска Docker состоит из активации системного демона (серверной части) и проверки связи между клиентом и сервером.

### 1. Запуск системной службы

В Arch Linux после установки пакет Docker не запускается автоматически. Для управления жизненным циклом службы используется системный менеджер `systemctl`.

* **Запуск:** `sudo systemctl start docker` (запускает демон в текущей сессии).
* **Автозагрузка:** `sudo systemctl enable docker` (позволяет Docker запускаться автоматически при старте системы).

### 2. Архитектурная проверка: `docker version`

Команда `docker version` является основным инструментом диагностики. Она подтверждает работоспособность клиент-серверной модели.

**Результат вывода делится на два блока:**

1. **Client (КЛИЕНТ):**
* **Сущность:** Бинарный файл `/usr/bin/docker`.
* **Статус:** Работает всегда, когда вы вызываете команду. Он отвечает за парсинг ваших команд и их отправку через сокет.


2. **Server (СЕРВЕР/ДВИЖОК):**
* **Сущность:** Процесс `dockerd` (демон).
* **Статус:** Должен быть активен (running). Если демон не запущен, клиент выдаст ошибку: *"Cannot connect to the Docker daemon"*.



### 3. Механика взаимодействия (The Hard Way)

Связь между клиентом и сервером осуществляется через **Unix-сокет**: `/var/run/docker.sock`.

* Если при выполнении команды без `sudo` вы получаете ошибку *Permission Denied*, это означает, что у вашего пользователя нет прав на запись в этот файл сокета.
* **Решение для Arch:** Добавление пользователя в группу `docker`:
`sudo usermod -aG docker $USER` (требуется перезагрузка сессии).

> [!IMPORTANT] Безопасность
> Доступ к сокету Docker эквивалентен правам `root` в системе, так как позволяет запускать контейнеры с монтированием хостовой файловой системы.
