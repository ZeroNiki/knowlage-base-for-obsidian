# Хранение данных в Docker: Writable Layer vs Volumes

В Docker используется концепция разделения неизменяемых данных образа и динамических данных работающего контейнера.

### 1. Слой записи (Writable Layer)

По умолчанию Docker использует стратегию **Copy-on-Write (CoW)**. Когда контейнер запускается, поверх неизменяемых слоев образа накладывается тонкий временный слой, доступный для записи.

* **Жизненный цикл:** Данные в этом слое неразрывно связаны с жизненным циклом конкретного экземпляра контейнера.
* **Удаление:** При выполнении команды `docker rm` этот слой полностью уничтожается вместе со всеми изменениями.
* **Производительность:** Запись во временный слой медленнее, чем прямая запись на диск хоста, из-за работы драйвера файловой системы (например, `overlay2`).

### 2. Тома (Volumes) и Bind Mounts

Механизм «прорезания дыры» в изоляции контейнера позволяет вынести данные за пределы его эфемерной файловой системы на физический диск хост-машины (например, вашего Arch Linux).

* **Персистентность:** Данные сохраняются даже после удаления контейнера. Новый контейнер может подключиться к тому же тому и продолжить работу.
* **Изоляция:** Тома позволяют обходить ограничения Copy-on-Write, обеспечивая производительность на уровне нативной файловой системы.
* **Типы монтирования:**
* **Bind Mounts:** Прямое монтирование конкретной папки или файла из хост-системы (например, `-v /home/user/project:/app`). Идеально для разработки.
* **Named Volumes:** Управляются самим Docker (обычно хранятся в `/var/lib/docker/volumes/`). Более безопасны и переносимы.


### 3. Сравнительная таблица

| Характеристика | Writable Layer | Volumes / Bind Mounts |
| --- | --- | --- |
| **Сохранность** | Удаляется вместе с контейнером | Сохраняется после удаления |
| **Место хранения** | Внутри области управления Docker | На файловой системе хоста |
| **Производительность** | Ниже (из-за OverlayFS) | Нативная (высокая) |
| **Основная цель** | Временные файлы, логи сессии | Базы данных, конфиги, код |

### 4. Синтаксис запуска

Для создания связи используется флаг `-v` или `--mount`:

```bash
# Bind Mount: связь локальной папки /tmp с папкой nginx в контейнере
docker run -d -v /tmp/html:/usr/share/nginx/html nginx

```

> [!CAUTION] Важное замечание
> Если вы монтируете папку с хоста в контейнер, где уже есть данные по этому пути, содержимое контейнера будет "скрыто" содержимым вашей локальной папки на время работы монтирования.
