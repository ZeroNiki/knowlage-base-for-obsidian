# Docker и SELinux: Мандатное управление доступом

> [[Linux namespaces]]

Если **Namespaces** создают для контейнера иллюзию владения собственной системой (ограничивая область «видимости»), то **SELinux** (Security-Enhanced Linux) выступает в роли жесткого инспектора, который проверяет каждое действие процесса, основываясь на политиках безопасности.

### 1. Фундаментальный принцип

Основное различие между стандартными правами Linux и SELinux:

* **Изоляция видимости (Namespaces):** «Я не вижу другие процессы и файлы».
* **Изоляция доступа (SELinux):** «Я вижу файл, но мне запрещено его трогать на уровне ядра, какие бы права (`root`) у меня ни были».

### 2. Механика меток (Labeling)

Система SELinux присваивает каждому объекту в системе (файлу, процессу, сетевому порту) специальную **метку безопасности (security context)**.

* **Процессы контейнера:** Обычно запускаются с меткой типа `svirt_lxc_net_t`.
* **Файлы на хосте:** Имеют свои метки (например, `user_home_t` для твоего домашнего каталога в Arch).
* **Логика проверки:** Когда процесс пытается прочитать файл, ядро Linux проверяет соответствие их меток. Если в политике SELinux нет правила «Процессу с меткой А разрешено читать файлы с меткой Б», доступ будет заблокирован, даже если процесс запущен от имени суперпользователя (`root`).

### 3. Роль в защите от Container Breakout

SELinux является критическим барьером в сценариях взлома:

1. **Сценарий атаки:** Злоумышленник находит уязвимость в приложении внутри контейнера и получает права `root`.
2. **Попытка побега:** Взломщик пытается получить доступ к чувствительным файлам хост-системы (например, `/etc/shadow` или конфигам Docker).
3. **Блокировка:** Поскольку метка процесса «злоумышленника» (контейнерная метка) не совпадает с меткой системных файлов хоста, ядро мгновенно пресекает попытку доступа.

### 4. Практика в Docker

В Docker работа с SELinux часто сводится к правильному монтированию томов (Volumes). Чтобы контейнер имел доступ к монтируемой папке на хосте с включенным SELinux, используются специальные флаги:

* **Флаг `:z`:** Docker автоматически изменит метку монтируемой папки, чтобы она была доступна **нескольким** контейнерам (shared).
* **Флаг `:Z`:** Docker назначит папке уникальную метку, доступную **только этому** конкретному контейнеру (private).

```bash
# Пример монтирования с автоматической установкой меток SELinux
docker run -v /var/db:/db:z my_app

```

> [!IMPORTANT] На заметку
> В твоем Arch Linux SELinux по умолчанию не всегда активен (часто используется AppArmor). Однако знание SELinux критично для работы с Red Hat, CentOS или Fedora, которые являются стандартом в корпоративной среде (DevOps).
