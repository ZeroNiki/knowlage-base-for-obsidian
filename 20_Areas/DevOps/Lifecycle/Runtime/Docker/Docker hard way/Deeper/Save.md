# Перенос данных: Docker Save vs Docker Export

В экосистеме Docker существует два принципиально разных способа упаковки данных в архив (`.tar`). Выбор метода зависит от того, что именно нужно сохранить: историю сборки и метаданные или текущий снимок файловой системы.

### 1. Docker Save (Работа с Образами)

Команда `docker save` предназначена для сохранения **образа** со всей его структурой.

* **Что сохраняется:** Все слои (layers), история изменений, теги и метаданные (переменные окружения, ENTRYPOINT, CMD).
* **Состояние:** Используется для **Images**.
* **Восстановление:** Команда `docker load`.
* **Применение:** Перенос образов между серверами без использования Registry (Docker Hub/GitLab Registry), создание полных бэкапов версий ПО.

### 2. Docker Export (Работа с Контейнерами)

Команда `docker export` создает плоский (flattened) архив текущего состояния **файловой системы контейнера**.

* **Что сохраняется:** Только текущие файлы. Вся история слоев и метаданные (ENTRYPOINT, переменные окружения) **удаляются**. Весь образ превращается в один единственный слой.
* **Состояние:** Используется для **Containers** (даже если они остановлены).
* **Восстановление:** Команда `docker import`.
* **Применение:** Создание базовых образов (например, кастомных сборок ОС), экспорт файловой системы для анализа, минимизация размера образа за счет удаления промежуточных слоев сборки.

### 3. Сравнительная таблица

| Характеристика | `docker save` | `docker export` |
| --- | --- | --- |
| **Объект** | Image (Образ) | Container (Контейнер) |
| **История слоев** | Сохраняется (каждый слой отдельно) | Удаляется (слои объединяются в один) |
| **Метаданные (ENV, CMD)** | Сохраняются | Теряются |
| **Команда импорта** | `docker load` | `docker import` |
| **Размер архива** | Обычно больше (из-за истории) | Обычно меньше (только текущие файлы) |

### 4. Примеры команд

**Сохранение и загрузка образа:**

```bash
# Сохранение образа в файл
docker save -o my_image.tar my_image:latest

# Загрузка обратно в Docker
docker load -i my_image.tar

```

**Экспорт и импорт контейнера:**

```bash
# Экспорт файловой системы контейнера
docker export my_container > my_container_fs.tar

# Импорт как нового образа (нужно заново указать метаданные)
cat my_container_fs.tar | docker import - my_new_image:latest

```

> [!TIP] Нюанс для DevOps
> Если вы используете `docker import`, помните, что вам придется заново прописывать `CMD` или `ENTRYPOINT` при запуске нового образа, так как импортированный архив — это просто "куча файлов" без инструкций по запуску.
