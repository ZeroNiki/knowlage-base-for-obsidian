# Различия: Docker Image vs Docker Container

В экосистеме Docker разделение на «образ» и «контейнер» аналогично разделению между «программой на диске» и «запущенным процессом в оперативной памяти».

### 1. Docker Image (Образ)

**Образ** — это инертный, неизменяемый (immutable) файл, который представляет собой моментальный снимок файловой системы и параметров запуска.

* **Природа:** Набор слоев, доступных только для чтения (**Read-Only Layers**).
* **Состояние:** Статическое. Образ нельзя «запустить» внутри него самого — его можно только использовать как основу.
* **Хранение:** Образы хранятся в реестрах (например, Docker Hub) или локально на диске хоста.
* **Аналогия:** Кулинарный рецепт или чертеж здания.

### 2. Docker Container (Контейнер)

**Контейнер** — это динамический, запущенный экземпляр образа. Он изолирован от других процессов в системе с помощью механизмов ядра Linux (Namespaces и Cgroups).

* **Природа:** Docker берет все Read-Only слои образа и добавляет поверх них один тонкий слой, доступный для записи (**Writable Layer** / Container Layer).
* **Состояние:** Динамическое. Контейнер имеет жизненный цикл (создан, запущен, остановлен, удален).
* **Изоляция:** Любые изменения, вносимые в работающем контейнере (создание файлов, установка софта), записываются только в верхний Writable Layer.
* **Аналогия:** Приготовленное блюдо по рецепту или возведенное здание по чертежу.

### 3. Механика Copy-on-Write (CoW)

Если процессу внутри контейнера нужно изменить файл, существующий в образе (Read-Only слой):

1. Docker копирует файл из нижнего слоя в верхний (Writable).
2. Изменения применяются к копии.
3. Оригинальный файл в образе остается нетронутым.

### 4. Сравнительная таблица

| Характеристика | Image (Образ) | Container (Контейнер) |
| --- | --- | --- |
| **Изменяемость** | Immutable (Неизменяем) | Mutable (Изменяем через Writable Layer) |
| **Слои** | Только Read-Only | Read-Only + 1 Writable Layer |
| **Назначение** | Хранение и доставка ПО | Выполнение и работа ПО |
| **Команды CLI** | `docker build`, `docker pull` | `docker run`, `docker stop`, `docker rm` |

> [!IMPORTANT] Ключевой вывод
> Один и тот же образ может быть основой для неограниченного количества контейнеров. При этом все они будут делить между собой Read-Only слои образа, что критически экономит место на диске.
