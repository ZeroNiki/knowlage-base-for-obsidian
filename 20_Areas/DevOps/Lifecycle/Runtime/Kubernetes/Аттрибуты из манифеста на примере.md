## **1️⃣ Deployment**

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: booking-service
```

* `apiVersion: apps/v1` — версия API Kubernetes, которая управляет объектами типа Deployment.
* `kind: Deployment` — объект Deployment создаёт и управляет **репликами подов**, обеспечивает обновление и масштабирование приложения.
* `metadata.name` — имя Deployment (`booking-service`), уникальное в namespace.

---

```yaml
spec:
  replicas: 1
```

* `spec.replicas` — сколько копий пода Kubernetes должен держать.
  В твоём случае: **1 экземпляр**.

---

```yaml
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
```

* `strategy.type` — стратегия обновления приложения.

  * `RollingUpdate` — обновляет приложение постепенно, не останавливая все поды одновременно.
  * `Recreate` — старые поды полностью удаляются, потом создаются новые.
* `rollingUpdate.maxSurge` — сколько новых подов может быть запущено **сверх нужного количества реплик** при обновлении. Здесь `1` значит: Kubernetes может создать **ещё 1 под** пока старый работает.
* `rollingUpdate.maxUnavailable` — сколько подов может быть **недоступно во время обновления**. `0` — значит все должны оставаться доступными.

> В комментарии есть закомментированный `Recreate`. Если заменить стратегию на `Recreate`, эти поля `rollingUpdate` игнорируются.

---

```yaml
  selector:
    matchLabels:
      app: booking-service
```

* `spec.selector.matchLabels` — как Deployment находит свои поды.
  Важно: метки должны совпадать с метками внутри `template.metadata.labels`.

---

```yaml
  template:
    metadata:
      labels:
        app: booking-service
```

* `template` — шаблон для создаваемых подов.
* `metadata.labels` — метки пода, которые Deployment использует для **связывания с селектором**.

---

```yaml
    spec:
      containers:
        - name: booking-service
          image: zeroniki/booking-service:7.0
          ports:
            - containerPort: 8083
```

* `spec.containers` — список контейнеров внутри пода (у тебя один).
* `name` — имя контейнера.
* `image` — Docker-образ для контейнера.
* `ports.containerPort` — порт внутри контейнера, на котором приложение слушает запросы.

---

### **Переменные окружения (env)**

Ключевые моменты:

```yaml
- name: POSTGRES_HOST
  valueFrom:
    configMapKeyRef:
      name: app-config
      key: POSTGRES_HOST
```

* `name` — имя переменной внутри контейнера.
* `valueFrom.configMapKeyRef` — берём значение из ConfigMap `app-config` по ключу `POSTGRES_HOST`.
* `valueFrom.secretKeyRef` — берём секрет из Secret, например `DB_PASSWORD` из `app-secrets`.
* `value` — прямое значение, например `reservations_db`.

Эта конструкция позволяет разделять **конфигурацию и секреты** от кода.

## **2️⃣ Service**

```yaml
apiVersion: v1
kind: Service
metadata:
  name: booking-service
```

* `apiVersion: v1` — базовая версия API для Service.
* `kind: Service` — объект, который делает поды доступными **снаружи или внутри кластера**.
* `metadata.name` — имя сервиса (`booking-service`).

---

```yaml
spec:
  type: NodePort
```

* `type`:

  * `ClusterIP` (по умолчанию) — доступ только внутри кластера.
  * `NodePort` — выделяет порт на **всех нодах кластера**, чтобы можно было обратиться к приложению извне.
  * `LoadBalancer` — для облака, выдаёт внешний IP.
  * `ExternalName` — делает алиас на внешний DNS.

---

```yaml
  selector:
    app: booking-service
```

* Связывает Service с подами, у которых есть **метка `app: booking-service`**.

---

```yaml
  ports:
    - name: http
      port: 8083
      targetPort: 8083
      nodePort: 30083
```

* `name` — имя порта (нужно, если несколько портов).
* `port` — порт **сервиса** внутри кластера.
* `targetPort` — порт, на который сервис проксирует внутри пода (контейнера).
* `nodePort` — порт на всех нодах кластера, через который сервис доступен извне. В твоём случае: `30083`.

---

 **Вывод:**

* **Deployment**: управляет подами, стратегиями обновления и масштабированием.
* **Service**: делает приложение доступным, связывает поды с внешним миром.
* **Env + ConfigMap/Secret**: позволяют безопасно передавать конфигурацию и пароли без изменения образа.
* **Стратегия обновления**: `RollingUpdate` обновляет постепенно, `Recreate` — полностью.
