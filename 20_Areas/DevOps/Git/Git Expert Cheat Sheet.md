#  Git Expert Cheat Sheet: От кодера к мастеру
> [!quote] Инженерный подход
> Git — это не просто сохранение кода, это **контентно-адресуемое хранилище** с интерфейсом файловой системы.

---

##  1. Фундаментальная Архитектура

### Снимки (Snapshots) vs Дельты
| Характеристика | Традиционные VCS (SVN) | Git (Снимки) |
| :--- | :--- | :--- |
| **Хранение** | Список изменений файлов | Снимки всей системы |
| **Локальность** | Нужен сервер для истории | Вся история локальна |
| **Целостность** | Возможна порча данных | SHA-1 гарантирует неизменность |

---

##  2. Три Дерева (The Three Trees)
Понимание `reset` невозможно без осознания этих зон:

1. **Working Directory**: Файлы «на диске», которые вы редактируете.
2. **Index (Staging Area)**: Снимок следующего коммита.
3. **HEAD**: Указатель на последний коммит текущей ветки.

### Диагностика статуса (`git status -s`)
> [!example] Чтение колонок
> - `??` — Untracked (не отслеживается).
> - ` A` — Added (в индексе).
> - ` M` — Modified (в индексе).
> - `M ` — Modified (только на диске).
> - `MM` — Изменен на диске **после** добавления в индекс.

##  3. Исправление и Перезапись истории

### Исправление последнего коммита
```bash
# Если забыли файл или опечатку
git add <file>
git commit --amend # Заменяет старый коммит новым
```

### Git Reset: Ядерные варианты
| Команда | HEAD | Index | Working Directory | Безопасность |
| :--- | :---: | :---: | :---: | :--- |
| `git reset --soft` |  |  |  | Безопасно |
| `git reset --mixed`|  |  |  | Средне |
| `git reset --hard` |  |  |  | **ОПАСНО** |

### Инспекция различий
- `git diff`: Work vs Index (что еще не добавлено).
- `git diff --staged`: Index vs HEAD (что пойдет в коммит).

---

##  4. Отладка через `git bisect`
*Бинарный поиск виновного коммита за $O(\log_2 n)$.*

1. `git bisect start`
2. `git bisect bad` (текущий коммит сломан)
3. `git bisect good <hash>` (последний рабочий коммит)
4. **Тестирование**: Git переключает версии → проверяем → `git bisect good` или `bad`.
5. `git bisect reset` (возврат в начало).

## ️ 5. Hooks и Submodules

> [!important] Hooks (Хуки)
> Скрипты в `.git/hooks`. Не передаются при `clone`. 
> - **Pre-commit**: Линтинг, тесты.
> - **Pre-receive**: Проверка политик на сервере.

> [!warning] Submodules
> Опасайтесь состояния **Detached HEAD** внутри подмодулей. Всегда проверяйте ветку перед коммитом в подмодуль.

##  Pro Tips от экспертов
- **Все есть объект**: Любой файл — это `blob`, папка — `tree`, фиксация — `commit`. Проверьте это командой: `git cat-file -p <hash>`.
- **Smart HTTP**: Современный стандарт передачи данных, обходящий файерволы.
- **CLI > GUI**: Настоящая магия (interactive rebase, filter-branch) доступна только в терминале.
