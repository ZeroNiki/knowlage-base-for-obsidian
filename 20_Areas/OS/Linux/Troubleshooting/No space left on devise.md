#  Issue: No space left on device (Metadata/Inodes exhaustion)
## Описание
> Парадоксальная ситуация: `df -h` показывает наличие места, но ОС блокирует запись с ошибкой «No space left on device». Проблема кроется в исчерпании **Inodes** (индексных дескрипторов) или удерживании удаленных файлов активными процессами.

## Контекст / Окружение
- **ОС:** Arch Linux (Rolling-release)
- **Проблема:** Файловая система достигла лимита метаданных, либо процесс удерживает «удаленный» файл.

---

##  Теория: Inodes и структура данных


*Каждый файл в Linux — это пара: **Inodes** (метаданные) + **Data Blocks** (содержимое).*
*Если Inodes закончились, система не может создать новый файл, даже если блоки данных пусты.*

---

## Ход расследования

### 1. Анализ Inodes (Метаданные)
Проверка заполнения таблицы дескрипторов:
```bash
df -i
```
*Если `IUse%` достигло 100% — система «слепа» к файлам, так как адресная книга заполнена.*

### 2. Поиск аномальных директорий
Ищем каталоги с огромным количеством мелких файлов, не вызывая I/O-шторм:
```bash
find /var -xdev -type d -size +100k -print
```

### 3. Поиск "Ghost Files" (Deleted but Open)
Если `df -i` в норме, ищем файлы, которые были удалены (`rm`), но все еще занимают место в памяти процесса:
```bash
lsof +L1
```
*Флаг `+L1` покажет файлы, где счетчик ссылок меньше единицы, но они открыты процессом.*

---

## Решение

### Для Arch Linux (очистка кэша)
```bash
# Удаление старых версий пакетов
sudo paccache -r
```

### При исчерпании Inodes
1. Удалить скопления мелких файлов (кэши, старые сессии).
2. Архивировать данные, если они нужны: `tar -czf archive.tar.gz /path/to/data`.

### При "Ghost Files"
Выяснить PID процесса из `lsof +L1` и отправить сигнал переоткрытия (или рестарт):
```bash
# Пример для nginx
sudo systemctl reload nginx
```

## ️ SRE Checklist (Предотвращение)

> [!important] Мониторинг
> Не ограничивайтесь `df -h`! Настройте алерты на `df -i` (порог 80%).

- [ ] **Logrotate:** Проверить конфиги в `/etc/logrotate.d/`.
- [ ] **Garbage Collection:** Убедиться, что приложения (Redis/PHP/др.) чистят свои сессии.
- [ ] **Monitoring:** Добавить в Prometheus/Grafana метрику `node_filesystem_files_free`.

> [!tip] Инсайт
> Перезапуск сервиса — самый быстрый способ освободить "зависшие" блоки, так как это принудительно закрывает все файловые дескрипторы процесса.
