# 🐛 Issue: SSH Connection Refused (Port 2222)
## Описание
> Соединение активно отклонено целевым хостом на уровне L4 (TCP Reset). В отличие от Timeout, это указывает на то, что запрос дошел до сервера, но не был принят сокетом.

## Контекст / Окружение
- **Компонент:** `sshd.service`, `ufw`
- **ОС:** Ubuntu 22.04 LTS
- **Целевой порт:** 2222 (после попытки смены стандартного порта 22)
- **Логи/Ошибка:**
```bash
ssh: connect to host remote_host port 2222: Connection refused
```

---

## Ход расследования

- [ ] **Шаг 1: Проверка статуса демона.** Убедиться, что `sshd` запущен и не упал из-за ошибки в конфиге.
  ```bash
  systemctl status ssh
  ```
- [ ] **Шаг 2: Проверка прослушиваемых портов.**
  Выяснить, на каком порту и интерфейсе реально висит служба.
  ```bash
  ss -tulpn | grep ssh
  ```
  *Критично:* Если вывод пуст или там только порт 22 — конфиг не применился.
- [ ] **Шаг 3: Аудит брандмауэра.**
  Проверить, разрешен ли входящий трафик на 2222.
  ```bash
  ufw status numbered
  ```
- [ ] **Шаг 4: Анализ системного журнала.**
  Поиск ошибок биндинга порта.
  ```bash
  journalctl -u ssh -n 50
  ```

---

## Решение

### 1. Корректировка конфигурации
Проверить `/etc/ssh/sshd_config` или `/etc/ssh/sshd_config.d/*.conf`.
```bash
# Убедиться в наличии:
Port 2222
```
Перед рестартом обязательно валидируем синтаксис:
```bash
sudo sshd -t
sudo systemctl restart ssh
```

### 2. Настройка сетевого периметра (UFW)
```bash
sudo ufw allow 2222/tcp
sudo ufw reload
```

> [!success] Финальная проверка
> `ssh -v -p 2222 user@remote_host` — соединение успешно, если пройдена фаза `SSH2_MSG_KEXINIT`.

---

## 🧠 Превентивные меры (Best Practices)

> [!info] Правило "Открытых дверей"
> Никогда не закрывай текущую SSH-сессию, пока не подтвердишь успешный вход в **новом** окне терминала.

- **Checklist перед изменениями:**
	1. Добавить новый порт в `ufw` **ДО** смены порта в конфиге.
	2. Всегда запускать `sshd -t` (test mode).
	3. Иметь доступ через консоль провайдера (VNC/IPMI) на случай блокировки.
	4. Использовать `ss -tulpn` для подтверждения Binding (привязки к 0.0.0.0).
