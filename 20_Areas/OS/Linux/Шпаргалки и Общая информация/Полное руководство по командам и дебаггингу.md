# 🛠️ System Administration: Command Line & Debugging

> [!quote] Философия "Banish your mouse"
> Терминал предоставляет ту скорость и точность, которые недоступны при использовании мыши, превращая автоматизацию рутины в искусство управления инфраструктурой.

---

## 📂 1. Навигация и манипуляция структурой

В Linux нет «дисков», есть только единое дерево `/`.

| Команда | Инженерный контекст | Pro-Tip |
| :--- | :--- | :--- |
| `pwd` | Контроль пути перед `rm` или `mv` | Всегда проверяй, где ты! |
| `ls -laR` | Видит всё: права, скрытые файлы, подпапки | `ls -ltr` — сортировка по времени (новые внизу) |
| `cd -` | Прыжок в предыдущую директорию | Незаменимо при работе в двух папках |
| `mkdir -p` | Создание цепочки папок одной командой | Основа скриптов автоматизации |
| `ln -s` | **Single Source of Truth**: символические ссылки | Позволяет не дублировать конфиги |

> [!info] Архитектурный выбор
> Иерархия: `/etc` (конфиги), `/var/log` (логи), `/home` (данные), `/tmp` (временные файлы).

## 🔍 2. Инспекция контента и Поиск

Дебаггинг — это чтение состояния системы.

### Мониторинг логов
* `tail -f /var/log/syslog` — слежка за системой в реальном времени.
* `less +G /var/log/auth.log` — открыть лог и сразу перейти в конец.

### Механизмы поиска
1. **Grep (Фильтр):** `grep -E "^Error" file.log` (Поиск строк, начинающихся с Error).
2. **Find (Диск):** `find /etc -name "*.conf"` (Медленно, но точно).
3. **Locate (База):** `locate nginx.conf` (Мгновенно, требует `updatedb`).

## ⚙️ 3. Конвейеры: "The Power of Pipes"

Конвейер (`|`) — способ сборки сложного инструмента из простых утилит.

**Классический отчет по логам:**
```bash
cat access.log | awk '{print $1}' | sort | uniq -c | sort -rn
# Извлечь IP -> Отсортировать -> Посчитать уникальные -> Показать топ

* **sed**: Потоковая правка конфигов: `sed -i 's/old/new/g' config.file`
* **wc -l**: Быстрый подсчет строк/объектов.
```

## 🧠 4. Процессы и Ресурсы

| Команда | Назначение |
| :--- | :--- |
| `htop` | Интерактивный мониторинг (CPU, RAM, Swap) |
| `ps aux` | Полный снимок состояний процессов |
| `systemctl status` | Проверка «здоровья» юнитов (должно быть `active: running`) |
| `kill -15` | **SIGTERM**: Вежливое завершение (сохранение данных) |
| `kill -9` | **SIGKILL**: Принудительная остановка (риск порчи данных) |

## 🛡️ 5. Безопасность и Права

**Числовая нотация:** `4` (Read), `2` (Write), `1` (Execute).
* `chmod 755` — Владелец может всё, остальные — только читать и запускать.
* `chmod 777` — **Дыра в безопасности**, избегай этого.

> [!important] Специальные биты
> * **SUID**: Запуск от имени владельца (нужен для `passwd`).
> * **Sticky Bit**: В папке `/tmp` никто не может удалить чужой файл.

## 🌐 6. Сетевой стек

* **Интерфейсы**: `ip a` (просмотр IP и состояния L2/L3).
* **Порты**: `ss -tulpn` (современная замена `netstat`, показывает кто слушает порты).
* **DNS**: `dig google.com` или проверка локальных заглушек в `/etc/hosts`.
* **Диагностика**: `curl -I http://localhost` — проверка HTTP-заголовков.



---

## 📦 7. Архивация (Backup)

* **tar**: `tar -cvf backup.tar /etc` (создать архив).
* **Сжатие**:
    * `gzip` (`-z`): Быстро, среднее сжатие.
    * `bzip2` (`-j`): Баланс для средних нагрузок.
    * `xz` (`-J`): **Золотой стандарт** для бэкапов (максимальное сжатие).

---

## ⚡ 8. Культура Bash: Скорость

* `Ctrl + R`: Поиск команды по истории (нажми и начни печатать).
* `Ctrl + A / E`: Прыжок в начало/конец строки.
* `sudo !!`: Выполнить предыдущую команду от имени root.
* `Tab / Double Tab`: Автодополнение (защита от опечаток).
