# 🐧 Идентификация пользователей и групп в UNIX/Linux
## 👤 Пользователи (Users)
Каждый пользователь в системе имеет уникальное имя и числовой идентификатор. Данные хранятся в файле `/etc/passwd`.

### Структура `/etc/passwd`
Запись представлена одной строкой из 7 полей, разделенных двоеточием:

`mtk:x:1000:100:Michael Kerrisk:/home/mtk:/bin/bash`

1. **Имя входа (Username):** Уникальный текстовый идентификатор.
2. **Зашифрованный пароль:**
    - `x` — пароль хранится в «теневом» файле (`/etc/shadow`).
    - _Пустое поле_ — пароль не требуется.
3. **UID (User ID):** Числовой идентификатор.
    - `0` — зарезервирован для **root**.
    - Современные системы используют 32-битные UID.
4. **GID (Group ID):** Идентификатор основной (первичной) группы пользователя.
5. **GCOS (Доп. информация):** Настоящее имя пользователя или комментарии.
6. **Домашний каталог:** Путь, в который пользователь попадает при входе.
7. **Оболочка (Shell):** Интерпретатор команд (например, `/bin/bash`).

## 👥 Группы (Groups)
Группы используются для управления доступом к файлам и ресурсам. Пользователь может принадлежать к нескольким группам одновременно (**POSIX.1-1990**).

### Структура `/etc/group`
Файл содержит информацию о составе групп: `название_группы:пароль:GID:список_пользователей`

- **Название:** Уникальное имя группы.
- **GID:** Числовой идентификатор группы.
- **Список пользователей:** Имена пользователей, входящих в группу (через запятую).

> [!NOTE] Первичная vs Вторичная группа
> **Первичная группа** прописывается в `/etc/passwd` (поле GID).
> **Вторичные (дополнительные) группы** прописываются в `/etc/group`.

---

## 🛡️ Привилегированный пользователь (Superuser)
В системе существует особый аккаунт с неограниченными правами.
- **Имя:** `root`
- **UID:** `0`
- **Возможности:**
    - Обход всех проверок разрешений.
    - Доступ к любому файлу.
    - Управление любыми процессами.

---

## 🔐 Безопасность паролей (`/etc/shadow`)
Для повышения безопасности зашифрованные хеши паролей выносятся из общедоступного `/etc/passwd` в файл `/etc/shadow`.

|**Файл**|**Доступ**|**Назначение**|
|---|---|---|
|`/etc/passwd`|Чтение разрешено всем|Информация о пользователях (имена, ID, пути)|
|`/etc/shadow`|Только `root`|Хранение хешей паролей и параметров истечения срока|

### Алгоритмы шифрования
- **DES:** Исторический стандарт (13 символов).
- **MD5:** Создает 128-битный хеш (34 символа в файле).
- **Современные:** SHA-256 / SHA-512 (используются в большинстве новых дистрибутивов).

---

## 🛠️ Полезные команды
- `id` — показать текущий UID и все GID пользователя.
- `whoami` — узнать текущее имя пользователя.
- `groups` — список групп, в которых состоит пользователь.
- `ls -l` — просмотр владельца и группы файла.
